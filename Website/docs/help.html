1 :: Getting Started

In this document, you'll find a detailed guide for getting, installing, and using the PureTween design and portfolio template.

PureTween is a clean, simple and elegant jQuery and php driven template that is very easy to navigate and features animated tile based links on the main page. It supports an unlimited number of tiles with unlimited color options. They can be linked to other pages on the site or used to display plain text or just images.

After purchasing the template, unpack and place the files and directories at the top level of your website, your installation is complete. The template comes with an index.php file which acts not only as a default page but as a shell, pulling in other pages of the site using ajax, as they are requested. The template supports an unlimited number of pages, which are required to be php files, and animates the page into this "shell" when requested.

The following are some important pages of the template worth pointing out:

Index.php

"Index.php" at the root level of the template's directory structure is the base of the entire template and all of its content. It simply fetches and renders "header.php" and "footer.php".

Without a url hash in the browser bar, "index.php" displays default content located inside "home.php".

Home.php

"Home.php" contains the home page content for the template which consists of animated tile based links. Each tile is made up of an html "span" element with 2 additionally nested elements, "p" and another "span". See instructionson how to add additional tiles.

Web Fonts

The following fonts made available by Google Web Fonts are utilized in this template:
1. Dosis (400, 500, 600, 700)
2. Rancho (400)
3. Quantico (400, 700)

The Documentation

To access this document quickly and easily from your site prior to deploying to production, ensure that the variable "enableF1_docs" is set to true inside "scripts/customize.js". If true, you can simply press the F1 button while viewing your site to pull up the documentation. Be sure to disable "enableF1_docs" prior to deploying your site to production for obvious reasons.

2 :: HTML Structure

This template has a fixed layout. At the top level are two div elements with id's of "wrapper" and "footer" respectively. The wrapper element encapsulates all main content on all pages of the template, while the footer element houses a fixed width 3 column 1 row grid.

A. WRAPPER

The wrapper element, on all pages of this template, contains a base div element with id "content". The "#content" div is declared at the bottom of "includes/header.php". Since "header.php" is a wrapper for all pages of the template, this setup allows you to have nothing more than raw html markup in all other pages of your site that are linked to via the navigation menu. For example, you can have a page called "services.php" with nothing more than the following inside. 

		
	<span id="page-title">Our Services</span>
	<span id="page-content"> A simple paragraph describing what we have to offer.</span>

B. FOOTER

Default content contained inside the footerconsist of a "latest projects" section, a custom built twitter feed widget, and footer contact info section. This content is found and can be modified within "/includes/footer.php".

3 :: CSS Structure

Styling for this template is contained within a single well organized CSS document located at "css/styles.css". At the top of this file is a CSS reset rule whose goal is to establish a standard for the way html by default is rendered in different browsers.

Following the reset rule are a few general rules that include a number of multi-use class style definitions. Below these general purpose classes are the style rules specific to shaping the look and feel of the template. To edit or adjust the look of any element, find it's class name or id within these rules and adjust or add to it's list of attributes.

Styles within "style.css" are defined in the following manner and are indented accordingly to represent a nested element. 

		
	/* === #CONTENT DIV === */
	#content
	{
		width:1000px; 
		height:auto; 
		padding-bottom:5px;
	}
	
		#child-element-of-content
		{
			width:1000px; 
			height:50px; 
			padding:0px;
		}

4 :: Javascript

Javascript, in the form of raw javascript and also in jQuery form, is used extensively within this template with scripts centrally located in the "scripts/.." directory at the top level of the directory structure. Within this directory are two additional directories that hold downloaded scripts associated with:

1. The jQuery library (v1.7.1)
2. jQueryUI effects only extension (v1.8.20)
3. prettyPhoto lightbox viewer (v3.1.4)

PrettyPhoto is a modal lightbox style jquery based content and media viewer.

Customizing

Javascript used to control the template is also located in "scripts/main.js" and in "scripts/customize.js". The latter is a much more user friendly file that you can open and any file editor and have the ability to customize basic template functionality. If you possess javascript knowledge and would like advanced customization, feel free to edit the code found in "main.js".

6 :: PHP Code Usage

This template, although javascript driven, is powered by the PHP scripting language. The header (logo and everything above) and the footer (twitter widget and surrounding content) are nothing more than dynamic php files located inside the "includes/..." directory. Using php, these two files are called or "included" into index.php where the html within them are rendered. Additionally, content pages for the template are required to be .php files to help encorage and facilitate the addition of more dynamically, possibly server side, generated content. For static content, simply place the static html markup in the your php file as it were an html file.

PHP code is also used to handle delivery of the contact form included in this template. The script is located in "actions/act_sendContact.php" and is a basic skeletion for you to defined how you wish the handle the form details on the server after is it validated and sent.

7 :: SEO

For the purposes of optimizing this template for Google and other search engines, robust use of url hashes is made in order to enable support for the template and its pages to be crawled and indexed by search engine crawlers. The navigation links to other pages of the template using nothing more than a hash fragment corresponding to the "link to" page's file name. For example: 

	
      <a href="#!mypage" rel="canonical">My Page</a>

The above is a typical link found in the navigation bar for the template. Rather than saying [href="mypage.php"], we use a hash fragment with the name of the intended file. This tells search engines that not only is this is an ajax application but it also instructs them to index the content found at the fragmented url (www.sitename.com/#!mypage).

8 :: How To Instructions

1. To add a new tile to the template home page:

Place the following code at the end of "home.php" with the file you want to link to as the "id" attribute of the wrapper "span" element. Omit "id" and add "not-clickable" as an additional class attribute to this element if you do not wish the present the tile as a link. Edit "TILE DISPLAY LABEL" with the text you want as the tile label (...or title). Similarly edit "TILE DISPLAY SUB LABEL" with the sub label you wish to appear. 

	
	<span class="center-tile" id="file_name_to_fetch">
		
	<p>
	<span class="tilelabel"> <span class="label-text">TILE DISPLAY LABEL</span></span>
	</p>
	<span class="tile-sub-text-2">TILE DISPLAY SUB LABEL</span>
	
	</span>

Then, add the following to the page.tilePageColorMap object in "script/customize.js". With tile color and file name properties. 

	
	,
	{
		color:	"#B41F24",
		page:	"page_4",
		image: 	""
	}

2. To add a new page to the site:

Simply copy an existing page and rename it. Then, to add a link to get to the page in the navigation menu, open "header.php" and add an anchor tag to the existing collection located inside the div element with id "_menu". The "href" attribute of the anchor tag should contain the (renamed) file to point to. To add a new tile to be linked to the new page, follow the steps documented above.

2. To edit background colors and images:

Open "scripts/customize.js" where there are definitions for both background colors and images for tiles, the footer, and the entire template. I have predefined 7 background image locations as variables that you can simply plugin where you wish to have the image as a repeating background (page, footer ... etc). Feel free to defined more backgrounds as you insert the associated image inside "images/backgrounds" directory. You leave either image or color empty and keep in mind that images will always overlay a background color definition.
